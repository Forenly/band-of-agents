## The premise
Most agents operate alone — locked inside one environment, framework, or workflow. Enterprise work is collaborative: agents need to pass context, coordinate tasks, recruit other agents, involve humans, and operate across long-running business processes. **Band** is a shared interaction layer where agents communicate, exchange structured context, coordinate actions, discover/recruit other agents, and work together **across frameworks**.

## The challenge — Build a Cross-Framework Multi-Agent System with Band
Build a multi-agent system where **at least 3 agents collaborate through Band** across planning, execution, review, decision-making, or task handoff.

- **Minimum requirement:** ≥3 agents collaborating **through Band**.
- **Meaningful Band usage:** collaboration must happen *through* Band — not a thin wrapper, final notification, or simple output channel. Band must be part of the actual collaboration layer (during the workflow, not only before/after).
- **Flexible scope:** internal ops, software development, regulated workflows, or another strong enterprise use case.

Make agent collaboration **visible, useful, and central** to the workflow.

### Example project ideas
- Product launch: research → planning → copywriting → review agents collaborate via Band.
- Software delivery: planner → engineer → tester → reviewer hand off across the dev lifecycle.
- Compliance review: policy, legal, risk, ops agents share context before a recommendation.
- Support escalation: triage → knowledge → resolution → manager-review on complex tickets.
- Procurement: analysis → vendor-risk → approval → reporting agents coordinate decisions.

## Judging criteria

2. **Presentation** — clarity of problem, agent roles, Band's coordination role, context/handoff flow, value created.

4. **Originality** — goes beyond chatbot/single-agent/linear automation; shows what's possible when agents discover each other, divide work, review, escalate, collaborate across frameworks.
